1 Patent Quality Indicators Construction

The dependent variables for patent quality are derived from the OECD Patent Quality Indicators database (2025 edition). While the database provides a wide array of metrics (forward citations, family size, and grant lag), this thesis focuses on two specific measures:

Composite Quality Index (quality_index_4): An unweighted average of four normalized components (forward citations, patent family size, radicalness, and number of technical fields).

Breakthrough Patents (breakthrough): A binary indicator identifying patents in the top 1% of the citation distribution within their respective technology-year cohort.

Because the OECD quality indicators are provided at the patent-application level without geographical identifiers, a multi-stage merging procedure is required to construct the country-year panel used in the analysis. The Quality Indicators file is joined with the OECD HAN (Harmonised Applicant Names) database using the unique application ID (appln_id). To assign each patent to a country, the HAN database is linked to the Harmonised Names (HARM_NAMES) database, allowing me to map each application to the Person_ctry_code (ISO country code) of the patent applicant. Patents are assigned to years based on their filing date. Finally, the patent-level data is aggregated to the country-year level. The quality_index_4 is averaged across all patents for a given country-year to create a mean quality score, while the breakthrough indicator is summed to calculate the total count of high-impact patents.

The following Python script (using the polars library for high-performance data manipulation) executes this pipeline:

"""
OECD Patent Quality Indicators
normalization, winsorization, and aggregation procedures
"""

import polars as pl
import os

OUTPUT_DIR = ""
os.makedirs(OUTPUT_DIR, exist_ok=True)
# for reproducing, change the file paths to correctly the downloaded data, 
# they're too large for me to make this self running
INDICATORS_FILE = "Data/202502_OECD_PATENT_QUALITY_EPO_INDIC.txt"
COHORTS_FILE = "Data/202502_OECD_PATENT_QUALITY_EPO_INDIC_COHORT.txt"
HAN_PATENTS_FILE = "Data/202502_HAN_PATENTS.txt"
HARM_NAMES_FILE = "Data/202502_HARM_NAMES.txt"

FINAL_PATENT_FILE = "OECD_patent_quality_final.parquet"
FINAL_COUNTRY_FILE = "OECD_country_year_final.csv"

WINSOR_VARS = ["family_size", "grant_lag", "bwd_cits", "npl_cits", "claims", "fwd_cits5", "fwd_cits7"]
COMPOSITE_VARS = ["generality", "originality", "radicalness", "quality_index_4", "quality_index_6"]

# load the data

def load_data():

    cohorts = pl.read_csv(COHORTS_FILE, separator="|", infer_schema_length=0)
    cohorts = cohorts.rename({c: c.strip() for c in cohorts.columns})

    indicators = pl.read_csv(INDICATORS_FILE, separator="|", infer_schema_length=10000)
    indicators = indicators.rename({c: c.strip() for c in indicators.columns})

    han = pl.read_csv(HAN_PATENTS_FILE, separator="|")
    han = han.rename({c: c.strip() for c in han.columns})
    han = han.rename({"Appln_id": "appln_id"})

    harm = pl.read_csv(HARM_NAMES_FILE, separator="|")
    harm = harm.rename({c: c.strip() for c in harm.columns})
    harm = harm.select(["HARM_ID", "Person_ctry_code"])

    han = han.join(harm, on="HARM_ID", how="left")

    return cohorts, indicators, han

# main function

def main():

    cohorts, indicators, han = load_data()

    merged = indicators.join(han, on="appln_id", how="left")

    # ensure breakthrough is binary
    if "breakthrough" in merged.columns:
        merged = merged.with_columns(
            pl.col("breakthrough").fill_null(0).cast(pl.Int8)
        )

    # aggregation to country/year

    agg_exprs = []

    # normalized indicators
    norm_cols = [c for c in merged.columns if c.endswith("_norm")]
    for c in norm_cols:
        agg_exprs.append(pl.mean(c).alias(f"avg_{c}"))

    # mean for composite indicator
    for c in COMPOSITE_VARS:
        if c in merged.columns:
            agg_exprs.append(pl.mean(c).alias(f"avg_{c}"))

    # breakthrough counts
    agg_exprs.extend([
        pl.sum("breakthrough").alias("breakthrough_count"),
        pl.count("appln_id").alias("patent_count"),
        (pl.sum("breakthrough") / pl.count("appln_id")).alias("pct_breakthrough")
    ])

    country_agg = (
        merged
        .filter(pl.col("Person_ctry_code").is_not_null())
        .group_by(["Person_ctry_code", "filing"])
        .agg(agg_exprs)
        .rename({"Person_ctry_code": "iso3c", "filing": "year"})
        .sort(["iso3c", "year"])
    )

    country_agg.write_csv(
        os.path.join(OUTPUT_DIR, FINAL_COUNTRY_FILE)
    )

    merged.write_parquet(
        os.path.join(OUTPUT_DIR, FINAL_PATENT_FILE)
    )

if __name__ == "__main__":
    main()

2 Data Construction and Variable Assembly

In this section, I assemble the final panel dataset by merging the OECD Patent Quality Indicators with macroeconomic and financial development data.

2.0.1 Dependent Variables

I construct two primary dependent variables: 1. Patent Quality (quality_index_100): The raw OECD composite quality index is scaled to a 0-100 range for ease of interpretation.

  1. Patent Quantity (ln_patents_pc): patenting intensity is measured as patents per capita (scaled per million residents). This is then \(\log(x+1)\) transformed to account for right-skewness and potential zero values.

2.0.2 Independent Variables & Mean-Centering

To accurately capture the interaction between financial structure and intellectual property rights, I apply a \(\log(x+1)\) transformation to our financial development variables (Bank Credit, Market Capitalization, and Venture Capital as a % of GDP) due to their right-skewness.

Both the Park Index (IPR) and the logged Financial Development variables are mean-centered prior to interaction. This ensures that the main effect coefficients in the regressions represent the marginal effect evaluated at the sample mean, significantly improving the interpretability of the final models.

3 Baseline Two-Way Fixed Effects (TWFE) Models

In this section, I estimate two-way fixed effects (TWFE) panel regressions to establish baseline relationships between financial structure, intellectual property rights, and my two distinct innovation outcomes: Patent Quality and Patent Quantity.

3.0.1 Model Specification and Methodological Choices

For each of the three financial development channels (Market Capitalization, Bank Credit, and Venture Capital), I estimate models sequentially introducing macroeconomic controls.

The baseline equation takes the form: \[Y_{it} = \beta_1 FD_{it}^c + \beta_2 IPR_{it}^c + \beta_3 (FD_{it}^c \times IPR_{it}^c) + \mathbf{X'}_{it}\beta + \alpha_i + \lambda_t + \epsilon_{it}\]

Features of this approach: 1. Centered Variables (\(^c\)): As established in the data preparation phase, both the financial development variable (\(FD\)) and the Park Index (\(IPR\)) are mean-centered. This ensures that \(\beta_1\) and \(\beta_2\) are interpreted as the marginal effects of finance and IPR exactly at the sample mean of the other variable, rather than at an unrealistic zero value. 2. Fixed Effects (\(\alpha_i, \lambda_t\)): I include country fixed effects to absorb time-invariant institutional or cultural characteristics, and year fixed effects to control for global macro shocks and aggregate patenting trends. 3. Clustered Standard Errors: To account for serial correlation within countries over time, all standard errors are clustered at the country level using vcovHC.

# master function to run TWFE models and output HTML tables
run_twfe_incremental <- function(depvar, fd, fd_x_ipr, title) {

  # filter missing values specific to the target variables to maximize sample size
  base <- df %>%
    filter(
      !is.na(.data[[depvar]]),
      !is.na(.data[[fd]]),
      !is.na(ipr_c)
    )

  # convert to panel dataframe
  panel <- pdata.frame(base, index = c("iso3c", "year"))
  
  # define sequential formulas
  f1 <- as.formula(paste(depvar, "~", fd, "+ ipr_c +", fd_x_ipr))
  f2 <- update(f1, . ~ . + ln_gdp_pc)
  f3 <- update(f2, . ~ . + ln_trade)
  f4 <- update(f3, . ~ . + ln_rd)
  f5 <- update(f4, . ~ . + ln_tertiary)

  # estimate within-country, two-way models
  m1 <- plm(f1, data = panel, model = "within", effect = "twoways")
  m2 <- plm(f2, data = panel, model = "within", effect = "twoways")
  m3 <- plm(f3, data = panel, model = "within", effect = "twoways")
  m4 <- plm(f4, data = panel, model = "within", effect = "twoways")
  m5 <- plm(f5, data = panel, model = "within", effect = "twoways")
  
  # format labels
  fin_label <- switch(fd,
    "ln_market_c" = "Log Market Cap (Centered)",
    "ln_bank_c"   = "Log Bank Credit (Centered)",
    "ln_vc_c"     = "Log Venture Capital (Centered)"
  )
  
  # output formatted HTML table for this document
  html_table <- capture.output(
    stargazer(
      m1, m2, m3, m4, m5,
      type = "html",
      title = title,
      dep.var.caption = ifelse(
        depvar == "quality_index_100",
        "Dependent Variable: Patent Quality (0–100)",
        "Dependent Variable: Log Patents per Capita"
      ),
      covariate.labels = c(
        fin_label,
        "IPR Index (Centered)",
        paste(fin_label, "x IPR"),
        "Log GDP per Capita",
        "Log Trade Openness",
        "Log R&D Expenditure",
        "Log Tertiary Enrollment"
      ),
      se = list(
        cluster_se(m1), cluster_se(m2), cluster_se(m3), cluster_se(m4), cluster_se(m5)
      ),
      omit.stat = c("ser", "f"),
      notes = c(
        "Country and year fixed effects included in all models.",
        "Standard errors clustered by country."
      ),
      notes.append = TRUE,
      no.space = TRUE,
      header = FALSE 
    )
  )

  # print HTML table to the Quarto render
  cat(paste(html_table, collapse = "\n"))
  cat("<br><br>") # Add spacing between tables

  invisible(list(m1, m2, m3, m4, m5))
}

4 Zero-Inflated Negative Binomial (ZINB) Models

In this supplementary analysis, I shift the focus from average patent quality and volume to the extreme right tail of the innovation distribution: breakthrough patents.

4.0.1 Methodological Justification

Breakthrough innovations are inherently rare, meaning the breakthrough_count variable is strictly non-negative count data characterized by two econometric challenges:

  1. Excess Zeros: Most country-year observations record zero breakthrough patents.

  2. Overdispersion: The variance of breakthrough counts significantly exceeds the mean, violating the assumption of standard Poisson regression.

To address this, I estimate Zero-Inflated Negative Binomial (ZINB) models. The ZINB framework simultaneously estimates two equations:

  • The Selection (Zero-Inflation) Model: A logistic regression predicting the probability that a country is in a “certain zero” state.

  • The Count Model: A negative binomial regression estimating the actual number of breakthrough patents produced, conditional on the country clearing the zero-hurdle, while explicitly modeling the overdispersion.

4.0.2 Model Specification

Just as in the baseline models, I utilize mean-centered financial development (\(FD_{it}^c\)) and institutional quality (\(IPR_{it}^c\)) variables. The models below iterate through the financial channels (Market Capitalization, Bank Credit, and Venture Capital), sequentially adding macroeconomic controls.

(Note: Because ZINB models generate coefficients for both the count and zero-inflation equations simultaneously, the raw model outputs are provided below. Formatted, consolidated tables are available in the main text of the thesis).

# function to run ZINB and output clean HTML tables
run_zinb_incremental <- function(finance_var, title_name) {

  # match exactly with the variable names 
  fd_c  <- paste0(finance_var, "_c")
  fd_x  <- paste0(finance_var, "_x_ipr") 

  controls <- c("ln_gdp_pc", "ln_trade", "ln_rd", "ln_tertiary")
  models <- list()

  # 1. estimate Models
  for (i in 1:4) {
    intensity_controls <- paste(controls[1:i], collapse = " + ")

    formula_str <- paste0(
      "breakthrough_count ~ ",
      fd_c, " + ipr_c + ", fd_x,
      " + ", intensity_controls,
      " + offset(ln_internal_patents) | ln_gdp_pc"
    )

    models[[i]] <- zeroinfl(
      as.formula(formula_str),
      data = df,
      dist = "negbin"
    )
  }

  # 2. output HTML Tables
  cat("##", title_name, "\n\n")

  for (j in 1:4) {
    cat("### Model", j, "\n\n")
    model_sum <- summary(models[[j]])

    # calculate Log Theta explicitly for the HTML table
    m <- models[[j]]
    if (!is.null(m$theta) && !is.null(m$SE.theta)) {
      lt_est <- log(m$theta)
      lt_se  <- m$SE.theta / m$theta
    } else {
      lt_est <- NA
      lt_se  <- NA
    }

    # Count model
    count_coefs <- model_sum$coefficients$count
    if (!is.na(lt_est)) {
      count_coefs <- rbind(count_coefs, "Log Theta" = c(lt_est, lt_se, NA, NA))
    }

    count_table <- data.frame(
      Variable = rownames(count_coefs),
      Coefficient = round(count_coefs[,1], 3),
      Std_Error = round(count_coefs[,2], 3)
    )

    cat("#### Intensity (Negative Binomial Count Equation)\n\n")
    print(knitr::kable(count_table, row.names = FALSE))
    cat("\n\n")

    # selection model
    zero_coefs <- model_sum$coefficients$zero
    zero_table <- data.frame(
      Variable = rownames(zero_coefs),
      Coefficient = round(zero_coefs[,1], 3),
      Std_Error = round(zero_coefs[,2], 3)
    )

    cat("#### Selection (Logit Structural Zero Equation)\n\n")
    print(knitr::kable(zero_table, row.names = FALSE))
    cat("\n\n")

    cat("**Observations:**", models[[j]]$n, "\n\n")
    cat("---\n\n") 
  }

  invisible(models)
}

7.1 Joint Significance Tests for Interaction Models

To confirm that these variables meaningfully impact our innovation outcomes, I perform joint significance (Wald) tests. Specifically, I test the null hypothesis that the main effect and the interaction term are jointly equal to zero (\(H_0: \beta_{Main} = \beta_{Interaction} = 0\)).

# custom function to execute and format robust joint significance tests
test_joint_significance <- function(model, var_main, var_interaction, test_name) {
  cat(paste0("====================================================\n"))
  cat(paste0("Joint Test: ", test_name, "\n"))
  cat(paste0("H0: ", var_main, " = 0 AND ", var_interaction, " = 0\n"))
  cat(paste0("====================================================\n"))
  
  # execute linear hypothesis test using clustered standard errors
  wald_result <- linearHypothesis(
    model, 
    c(paste0(var_main, " = 0"), paste0(var_interaction, " = 0")), 
    vcov. = vcovHC(model, type = "HC1", cluster = "group")
  )
  
  print(wald_result)
  cat("\n\n")
}

run_trend_incremental <- function(depvar, fd, fd_x_ipr, title) {

  # maximize samples by finance type
  base <- df %>%
    filter(
      !is.na(.data[[depvar]]),
      !is.na(.data[[fd]]),
      !is.na(ipr_c)
    )

  # ensure year is numeric so fixest treats iso3c[year_numeric] as a linear trend
  base$year_numeric <- as.numeric(as.character(base$year))
  
  # specify the fixed effects and trend part of the formula
  fe_trend <- "| iso3c + year + iso3c[year_numeric]"
  
  # build specifications
  f1 <- as.formula(paste(depvar, "~", fd, "+ ipr_c +", fd_x_ipr, fe_trend))
  f2 <- as.formula(paste(depvar, "~", fd, "+ ipr_c +", fd_x_ipr, "+ ln_gdp_pc", fe_trend))
  f3 <- as.formula(paste(depvar, "~", fd, "+ ipr_c +", fd_x_ipr, "+ ln_gdp_pc + ln_trade", fe_trend))
  f4 <- as.formula(paste(depvar, "~", fd, "+ ipr_c +", fd_x_ipr, "+ ln_gdp_pc + ln_trade + ln_rd", fe_trend))
  f5 <- as.formula(paste(depvar, "~", fd, "+ ipr_c +", fd_x_ipr, "+ ln_gdp_pc + ln_trade + ln_rd + ln_tertiary", fe_trend))

  # run models with fixest (cluster = ~iso3c applies HC1 clustered standard errors)
  m1 <- feols(f1, data = base, cluster = ~iso3c)
  m2 <- feols(f2, data = base, cluster = ~iso3c)
  m3 <- feols(f3, data = base, cluster = ~iso3c)
  m4 <- feols(f4, data = base, cluster = ~iso3c)
  m5 <- feols(f5, data = base, cluster = ~iso3c)
  
  # format custom labels for the table
  var_dict <- c(
    quality_index_100 = "Patent Quality (0–100)", 
    ln_patents_pc = "Log Patents per Capita",
    ln_market_c = "Log Market Cap (Centered)",
    ln_bank_c = "Log Bank Credit (Centered)",
    ln_vc_c = "Log Venture Capital (Centered)",
    ipr_c = "IPR Index (Centered)",
    ln_market_x_ipr = "Log Market Cap x IPR",
    ln_bank_x_ipr = "Log Bank Credit x IPR",
    ln_vc_x_ipr = "Log Venture Capital x IPR",
    ln_gdp_pc = "Log GDP per Capita",
    ln_trade = "Log Trade Openness",
    ln_rd = "Log R&D Expenditure",
    ln_tertiary = "Log Tertiary Enrollment"
  )


  cat(paste0("### ", title, "\n\n")) 
  

  table_text <- capture.output(
    etable(
      m1, m2, m3, m4, m5,
      dict = var_dict,
      signif.code = c("***"=0.01, "**"=0.05, "*"=0.1),
      fitstat = ~ n + r2
    )
  )
  

  cat("```text\n")
  cat(paste(table_text, collapse = "\n"))
  cat("\n```\n")
  

  cat("\n*Notes: Country and year fixed effects included. Country-specific linear time trends included. Standard errors clustered by country.*\n")
  cat("<br><br>\n\n") 

  invisible(list(m1, m2, m3, m4, m5))
}

8 Variable Glossary

Source for PQI Data Source for WDI data Source for VC data Source for Park Index

8.0.1 Dependent Variables

  • quality_index_100 - The OECD composite patent quality index (avg_quality_index_4), scaled to a 0–100 range for interpretability.
  • patents_per_capita - Patent applications by residents, scaled per million people.
  • ln_patents_pc - Log-transformed patenting intensity (\(\log(x+1)\) of patents_per_capita). This replaces previous researcher-scaled metrics to better capture output while maximizing sample size.
  • breakthrough_count - The absolute count of breakthrough patents (top 1% most cited) produced by a country in a given year. Used exclusively as the dependent variable in the ZINB models.

8.0.2 Financial Development Variables (Independent)

  • bank_credit_gdp - Domestic credit provided by the financial sector (% of GDP). Proxy for traditional banking depth.
  • market_cap_gdp - Market capitalization of listed domestic companies (% of GDP). Proxy for public equity market depth.
  • VC_investment_pct - Venture capital investment as % of GDP (rescaled by \(\times 100\) to match the percentage units of the other financial variables).
  • ln_bank, ln_market, ln_vc - Log-transformed (\(\log(x+1)\)) versions of the respective financial metrics to address right-skewness.
  • ln_bank_c, ln_market_c, ln_vc_c - Mean-centered versions of the log-transformed financial variables. Centering ensures that the main effects in the interaction models are evaluated at the sample mean.

8.0.3 Institutional Variables (Independent)

  • ipp_index_IP_transformed - The Ginarte-Park Intellectual Property Protection Index (0–5 scale), using a standard five-year fill interpolation for missing inter-period years.
  • ipr_c - Mean-centered Park Index.

8.0.4 Interaction Terms

  • ln_bank_x_ipr, ln_market_x_ipr, ln_vc_x_ipr - The interaction between the mean-centered financial variable and the mean-centered Park Index ( ln_market_c \(\times\) ipr_c).

8.0.5 Macroeconomic Controls

Note: All continuous macro controls are \(\log(x+1)\) transformed to capture proportional relationships and minimize the influence of extreme outliers. - ln_gdp_pc - Log of real GDP per capita (constant 2015 USD). - ln_trade - Log of trade openness (Imports + Exports as % of GDP). - ln_rd - Log of Gross Domestic Expenditure on R&D (% of GDP). - ln_tertiary - Log of gross tertiary school enrollment (%).
